This Privacy & Data Handling Policy describes how DeepAudit Inc. ("DeepAudit," "we," "us," or "our") collects, processes, stores, and protects information when you use the DeepAudit Intelligence Platform ("Service"). We are committed to handling your data with the highest standard of care and transparency.

1. What Data We Collect

Repository Code (Temporary)

Source code from repositories submitted for audit. This data is cloned to ephemeral storage for the sole purpose of performing the requested analysis and is automatically deleted within 24 hours of audit completion.

Git Metadata

Repository structure, commit history, branch information, contributor statistics, and file metadata necessary for comprehensive audit analysis.

Account Information

Email address, name, and company information provided during account creation or audit submission. Used for audit delivery, account management, and service communications.

Usage Data

Service interaction data including audit requests, API usage, and platform access logs. Used for service improvement, capacity planning, and security monitoring.

2. How We Process Your Code

Your code follows a strict, auditable processing pipeline with no persistent storage:

1

Clone

Repository is cloned to encrypted ephemeral storage within an isolated compute environment. No persistent disk is used.

2

Analyze

Our multi-phase AI engine scans the codebase across 40+ categories, generating structured findings with severity ratings, affected files, and remediation recommendations.

3

Generate Signals

Audit findings are compiled into structured reports. Only the findings (not the source code) are persisted in the report database.

4

Delete

All source code is automatically and permanently deleted from ephemeral storage within 24 hours of audit completion. Deletion is verified and logged.

4. Security & Encryption

DeepAudit employs enterprise-grade security measures at every layer of the processing pipeline:

In Transit

All data in transit is encrypted using TLS 1.3 with forward secrecy. API endpoints enforce HTTPS. No plaintext transmission of code or credentials.

At Rest

Ephemeral storage is encrypted using AES-256 encryption. Encryption keys are managed via AWS KMS with automatic rotation.

Isolation

Each audit runs in an isolated compute environment. No cross-customer data access. Network-level isolation between processing jobs.

Access Control

Role-based access control (RBAC) for all internal systems. Multi-factor authentication required for infrastructure access. Audit logging on all administrative actions.
