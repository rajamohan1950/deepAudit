Single Point of Failure Map

Infrastructure SPOF

CRITICAL

Single PostgreSQL instance — no read replicas, no failover

Blast radius: Complete platform outage if primary database fails. All 3 services depend on this single instance. RTO undefined — no automated recovery procedure exists.

Evidence: infra/terraform/rds.tf:23 — multi_az = false, no read_replica resource defined

Code SPOF

CRITICAL

Payment processing in monolithic service — no circuit breaker

Blast radius: A Stripe API timeout cascades through the entire order pipeline. No timeout, retry, or circuit-breaker pattern. A 30-second Stripe outage queued 12K requests in load testing.

Evidence: payment-service/src/checkout.py:234 — synchronous stripe.Charge.create() with no timeout parameter

Key Signal Examples

Detailed evidence for selected findings. Click to expand.

Signal

Database connection string with username and password hardcoded directly in source. Credentials have not been rotated since initial commit (14 months ago). Same credentials used across staging and production environments.

Evidence

payment-service/src/db/connection.py:47 — DB_URL = "postgresql://shopflow_admin:Pr0d_Sh0p!2024@prod-db.internal:5432/shopflow"

Failure Scenario

Any developer, contractor, or compromised CI runner with repository access has full database admin privileges. Credential exposure in a breach would grant complete data exfiltration capability including PII and payment records.

Remediation

Migrate to AWS Secrets Manager or HashiCorp Vault. Rotate all credentials. Implement least-privilege DB roles.

Signal

Login, password reset, and MFA verification endpoints have no rate limiting middleware. An attacker can attempt unlimited credential combinations without throttling or account lockout.

Evidence

api-gateway/routes/auth.ts:112 — router.post('/login', authController.login) — no rateLimiter middleware in chain

Failure Scenario

Credential stuffing attack using leaked credential lists. With ~15K enterprise accounts, a successful breach of even one admin account could expose all tenant data via the admin API.

Remediation

Add token-bucket rate limiter (e.g. express-rate-limit). Implement progressive lockout after 5 failed attempts. Add CAPTCHA after 3 failures.

Signal

The product recommendation endpoint passes user-supplied text directly to the ML model without input validation, size limits, or sanitization. The model accepts freeform text queries that could be crafted to manipulate recommendation outputs.

Evidence

ml-service/app/predict.py:89 — model.predict(request.body["query"]) — no schema validation, no input length limit

Failure Scenario

Prompt injection could manipulate recommendation rankings (promoting competitor products). Oversized inputs could cause OOM kills on the inference server. Adversarial inputs could extract training data.

Remediation

Add Pydantic input schema with max_length. Implement input sanitization layer. Add output filtering to prevent data leakage.

Signal

Product listing endpoint issues 1 query to fetch products, then N individual queries to fetch categories, images, and pricing for each product. With a default page size of 50, this produces 151 database queries per request.

Evidence

api-gateway/src/resolvers/products.ts:67 — products.map(async p => ({ ...p, category: await getCategory(p.categoryId) }))

Failure Scenario

Under peak traffic (Black Friday), p95 latency exceeds 2 seconds. Database connection pool exhaustion under concurrent load. Poor mobile UX driving cart abandonment.

Remediation

Implement eager loading with JOIN queries. Add DataLoader for batching. Consider materialized views for product listings.

Signal

JWT access tokens are stored in localStorage instead of httpOnly cookies. Combined with 3 unsanitized user-input rendering points in the storefront, an XSS attack can exfiltrate session tokens for any logged-in user.

Evidence

storefront/src/auth/store.ts:12 — localStorage.setItem('access_token', response.token)

Failure Scenario

Any XSS vector (product description, review, search) allows attacker to steal admin session tokens. With admin tokens, attacker has full platform access including customer PII, payment data, and store configuration.

Remediation

Migrate to httpOnly, Secure, SameSite=Strict cookies. Implement CSP headers. Sanitize all user-generated content rendering.
